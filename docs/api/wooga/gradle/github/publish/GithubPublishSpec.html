Package: wooga.gradle.github.publish

[Groovy] Interface GithubPublishSpec

    • wooga.gradle.github.base.GithubSpec
    • All Superinterfaces:
      GithubSpec

      Task spec definition for a github publish task/action. https://developer.github.com/v3/repos/releases/#create-a-release

      All set methods come in two flavors.

      • a set assignment method setTagName(...)
      • a mutator method tagName(...) Both return the current object. The provided value can be any object with a valid conversion method available (toString, toBoolean, etc). If the value is a Closure or Callable and the docs doesn't state otherwise, the result after calling the object and executing the matching conversion method on the return value will be used. All values will be evaluated when calling the get method. 
         String getReleaseName() {
         if(this.releaseName == null) {
             return null
         
         if (this.releaseName instanceof Callable) {
             return ((Callable) this.releaseName).call().toString()
         }
         this.releaseName.toString()
     }
 }

        • PublishMethod getPublishMethod()

          Returns a PublishMethod value indicating if a release should be created or updated. Potential values are * create * update * createOrUpdate This method parameter can be used to specify the intention of the publish operation. The GithubPublish will use the given @{code tagName} to find a release in the repository. When set to create, the GithubPublish task will fail if a release already exists. When set to update, the GithubPublish task will fail if a release doesn't exists. When set to createOrUpdate, the GithubPublish task will create a release if missing.

          Returns:
          PublishMethod indicating if a release should be created or updated.
          Default:
          PublishMethod.create
          See Also:
          GithubPublish
